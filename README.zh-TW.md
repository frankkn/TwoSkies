# TwoSkies ☔ 兩片天空

**一個畫面、兩片天空——下半是你的，上半是那個人的。唯一的互動，是一天一次的輕點：「我來看過你的天空了」。**

🔗 Live：**https://twoskies.web.app**

## 這是什麼

你站在台北的晴天裡，同一個畫面上，倫敦正對你在乎的人下著雨。

TwoSkies 建立在一個想法上：**不聊天，但知道彼此的天空**。所有被知道的事，都是自願說的——你在哪個城市（刻意只到城市級）、你今天有沒有來看。僅此而已。

還沒配對時，它就是一個安靜的單人天氣 app；有人兌換了你的邀請，畫面才長出第二片天空。

### 產品規則（不可妥協）

- **一個配對只有兩個人。** 一組邀請碼、一次兌換、一人同時只在一個配對裡
- **看是安靜的。** 打開 app 看對方的天空不留任何痕跡——沒有已讀回條，永遠沒有
- **打卡是主動的。** 每天可以輕點一次「我來看過你的天空了」，對方下次打開 app 時會看到淺淺一句「{稱呼}來看過你的天空了」。點了就點了，收不回；沒點就是沒點，不補、不提醒
- **「今天」以打卡者自己的時區為準**，跨半球也不會被日界線捉弄
- **不留歷史。** UI 永遠只顯示今天；打卡紀錄由 Firestore TTL 在 48 小時後刪除——「沒有歷史」是基礎設施兌現的承諾，不是 UI 假裝
- **任一方可以結束。** 解除配對即刻全刪——不需理由、不另行通知。關係的誠實包含結束
- **沒有聊天、沒有連續天數、沒有推播通知。** 知道對方來過的唯一方式，是你自己打開它

## 天氣

每片天空顯示即時天況（慢而安靜的動畫——雲移、雨絲、星光，太陽月亮跟著那座城市自己的晝夜）、目前溫度與今日高低溫、24 小時逐時列、附 iPhone 式溫度範圍條的 7 天預報——整組放在磨砂卡片上，時間一律用**那個地方的時鐘**。對方的晚上八點，就是你看見的他的夜。

天氣資料來自 [Open-Meteo](https://open-meteo.com)——免費、無 API key、瀏覽器直接查。

## 誠實是用基礎設施兌現的

產品的承諾由 Firestore Security Rules 強制，不靠自律：

- **座標一律粗化到 0.1°（約 10 公里）**——rules 用數學驗證；就算改造 client 也寫不進精確位置
- **每個文件都有欄位白名單**——沒辦法用多餘欄位夾帶出一個聊天室
- **打卡只能建立**，ID 帶伺服器時間戳；配對存續期間不可能刪除
- **邀請碼 24 小時內過期**（rules 封頂），且必須在加入配對的同一筆交易中原子性地銷毀
- **一人一配對**由 `getAfter` 交易性強制

以上全部有 33 個 emulator 測試覆蓋（`npm run test:rules`）。

## 技術

- **Vite + React + TypeScript + Tailwind CSS v4**
- **Firebase**：Google 登入、Firestore（`onSnapshot` 即時同步）、Hosting、App Check——完全沒有自建後端
- **Open-Meteo** 提供天氣與城市地理編碼

## 本地開發

```bash
npm install

# 純 UI 原型，假配對資料（不需要 Firebase）
VITE_USE_MOCK=1 npm run dev

# 完整功能，連本地 emulator
npm run emulators                  # Auth :9099 + Firestore :8080
VITE_USE_EMULATORS=1 npm run dev

# Security Rules 測試（33 個案例）
npm run test:rules
```

要連真實的 Firebase 專案，把 `.env.example` 複製成 `.env.local` 填入你的 web app 設定。

## 刻意不做的事

聊天、訊息、照片、已讀回條、連續天數、歷史紀錄、推播通知、精確定位、移動軌跡（只有「現在在哪」，沒有「去過哪」）、第三人、群組，以及任何 engagement 優化。

天氣一直都在，想看的人自然會來。
